# 面向航电智能化应用的 Agent-RL 融合系统方案

## 题目建议

推荐题目：

《面向航电智能化应用的分层强化学习自主决策 Agent 系统设计与实现》

备选题目：

- 《基于分层强化学习的飞行器自主决策 Agent 系统研究与实现》
- 《面向飞行任务仿真的多模块智能 Agent 系统设计与自主决策方法研究》
- 《面向航电智能体的飞行器自主决策与知识增强方法研究》

推荐采用第一或第二个题目。它们既保留了当前项目最扎实的分层强化学习核心，又能合理引入大语言模型智能体、RAG 知识增强和视景展示等扩展模块。

## 一、方案定位

当前项目已经具备比较完整的飞行器自主决策基础，包括 JSBSim 飞行动力学仿真、1v1/2v2 空战环境、PPO/MAPPO、自博弈、分层强化学习、导弹攻防和 Tacview 可视化。后续不建议推翻现有强化学习主线，而应将其封装为智能体系统中的核心决策模块。

新的毕设定位可以表述为：

面向航电智能化应用，构建一个以分层强化学习飞行器自主决策模型为核心，以大语言模型 Agent 为任务理解、知识检索、流程调度和解释生成中枢的多模块智能体原型系统，并在 JSBSim 仿真环境中完成闭环验证。

在这个定位下，语言模型不直接输出底层飞控动作，而是负责高层任务理解、流程编排、知识查询、决策解释和人机交互。实时控制和空战动作仍由已有的强化学习策略完成。

## 二、总体架构

系统采用“LLM Agent 编排 + RL 决策执行 + JSBSim 仿真闭环 + RAG 知识增强 + 视景展示”的架构。

```text
任务输入 / 人机交互
        |
        v
任务管理 Agent（LLM + LangGraph）
        |
        v
态势感知模块（读取 JSBSim 飞行状态、敌我态势、导弹状态）
        |
        v
安全约束模块（高度、速度、过载、发射条件、边界检查）
        |
        v
RL 决策模块（PPO / MAPPO / 分层强化学习模型）
        |
        v
执行接口模块（动作映射到 JSBSim env.step）
        |
        v
JSBSim 仿真环境（飞行动力学 + 导弹模型）
        |
        +------> 视景与可视化模块（Tacview / 轨迹 / 态势面板）
        |
        +------> 日志与评估模块（奖励、胜率、命中率、决策链路）

任务管理 Agent <------> RAG 知识模块（飞行规则、任务文档、装备参数、异常处置）
```

推荐主框架采用 LangGraph。其优势是可以把智能体过程显式建模为状态图，便于表达任务解析、RAG 查询、RL 决策、安全校验、执行反馈和日志记录等有状态流程，也便于论文中展示系统架构和决策链路。

RAG 模块可以使用 LlamaIndex 或 LangChain retriever 实现。考虑到毕设工作量，RAG 不需要一开始做复杂 Agentic RAG，只需完成知识库构建、检索、回答引用和辅助解释即可。

## 三、组成模块

| 模块 | 主要作用 | 推荐实现方式 | 与当前项目关系 |
|---|---|---|---|
| 任务管理 Agent | 理解任务目标，决定调用 RAG、RL 决策、安全检查或解释模块 | LangGraph 节点 + LLM 工具调用 | 新增系统外壳 |
| 态势感知模块 | 读取高度、速度、姿态角、敌我距离、AO/TA、导弹状态 | 封装现有 JSBSim 环境状态 | 复用环境层 |
| RL 决策模块 | 输出机动、规避、攻击、导弹发射等动作 | 封装现有 PPO/MAPPO/分层策略 | 核心算法贡献 |
| RAG 知识模块 | 检索飞行规则、装备参数、任务说明和异常处置 | LlamaIndex 或 LangChain retriever | 新增解释与知识增强能力 |
| 安全约束模块 | 检查高度、速度、过载、发射距离和攻击角 | 规则函数 + YAML/JSON 阈值配置 | 强化航电场景可信性 |
| 执行接口模块 | 将 Agent/RL 决策转为环境动作并推进仿真 | 调用 env.step 和现有 action 映射 | 复用仿真闭环 |
| 视景模块 | 展示轨迹、态势、奖励曲线和空战过程 | Tacview + 轨迹图 + 状态面板 | 基于已有渲染能力扩展 |
| 日志评估模块 | 记录任务、状态、动作、RAG 结果、奖励和胜负 | JSONL/CSV + 实验统计图 | 支撑论文实验 |

## 四、核心数据流

一次完整闭环可以设计为：

1. 用户输入任务或选择实验配置，例如“1v1 导弹攻防”“规避来袭导弹”“执行优势占位”。
2. 任务管理 Agent 将自然语言或配置解析为结构化任务目标。
3. 态势感知模块从 JSBSim 环境读取当前飞行状态和战场态势。
4. Agent 判断是否需要调用 RAG，例如查询导弹发射条件、任务规则或异常处置说明。
5. RL 决策模块根据 observation 输出机动动作或导弹发射动作。
6. 安全约束模块检查动作是否违反高度、速度、过载、攻击角、最小发射间隔等边界。
7. 执行接口模块将安全动作传入环境，推进仿真。
8. 日志模块记录状态、动作、奖励、风险标记、RAG 结果和解释文本。
9. 视景模块输出 Tacview 轨迹或状态面板，用于论文展示与行为分析。

建议定义三个最小数据结构：

```python
AgentState = {
    "task_goal": str,
    "scenario_name": str,
    "flight_state": dict,
    "rag_context": list,
    "rl_action": object,
    "safe_action": object,
    "episode_metrics": dict,
    "decision_trace": list,
}
```

```python
DecisionRequest = {
    "agent_id": str,
    "observation": list,
    "scenario": str,
    "mode": "combat | dodge_missile | shoot_missile | heading",
}
```

```python
DecisionResult = {
    "raw_action": object,
    "safe_action": object,
    "explanation": str,
    "risk_flags": list,
}
```

这些结构不要求一开始全部复杂实现，但可以作为系统接口和论文图表的基础。

## 五、建设计划表

| 阶段 | 建设内容 | 具体任务 | 产出 | 优先级 |
|---|---|---|---|---|

| 第二阶段 | Agent 编排层搭建 | 使用 LangGraph 建立任务解析、态势读取、决策调用、安全检查、执行反馈流程 | Agent 工作流原型 | 高 |
| 第三阶段 | RL 决策工具封装 | 将已有 actor 模型和 env step 封装为 DecisionTool | 可被 Agent 调用的决策模块 | 高 |
| 第四阶段 | RAG 知识模块 | 建立飞行规则、任务文档、导弹参数、异常处置知识库 | 知识检索与解释模块 | 中 |
| 第五阶段 | 安全约束模块 | 对低高度、过载、超距发射、攻击角异常等动作进行检查 | 安全过滤与风险标记 | 高 |
| 第六阶段 | 视景展示模块 | 输出 Tacview、轨迹图、奖励曲线和态势面板 | 可视化实验结果 | 中 |
| 第七阶段 | 评估与论文实验 | 对比纯 RL 与 Agent-RL 系统在成功率、生存率、命中率、解释性上的表现 | 实验表格和论文结论 | 高 |

## 六、论文创新点

可以将创新点组织为五个层次：

1. 提出面向航电智能化应用的多模块 Agent-RL 融合架构。语言模型负责任务理解、流程调度和知识增强，强化学习模型负责实时自主决策。
2. 将现有分层强化学习飞行器自主决策模型封装为智能体系统的核心决策模块，实现高层任务意图到低层飞行动作的闭环映射。
3. 引入 RAG 知识增强机制，使系统能够检索飞行规则、任务说明、导弹参数和异常处置知识，提高人机交互和决策解释能力。
4. 设计安全约束与风险标记模块，对强化学习输出动作进行边界检查，提升航电仿真场景下的可控性。
5. 基于 JSBSim 和 Tacview 完成仿真闭环验证，通过轨迹、奖励、胜负、命中率、生存率等指标分析系统效果。

## 七、测试与验收

| 测试类别 | 测试内容 | 验收标准 |
|---|---|---|
| 单模块测试 | RL 决策工具能加载模型并输出动作 | 动作 shape 与当前环境一致 |
| 工作流测试 | Agent 完成“状态读取 -> 决策 -> 安全检查 -> 执行”闭环 | 单步仿真无异常 |
| RAG 测试 | 输入任务问题后检索相关规则或参数 | 返回内容与知识库来源一致 |
| 安全测试 | 构造低高度、过载、超距发射等状态 | 系统能给出风险标记或拒绝动作 |
| 场景测试 | 1v1 无武器、1v1 导弹、2v2 场景 | 能完成 episode 并记录指标 |
| 论文实验 | 对比纯 RL 与 Agent-RL 系统 | 给出成功率、生存率、命中率和解释日志对比 |

## 结论

该方案的关键不是让大语言模型替代强化学习控制器，而是让语言模型成为系统级智能体的任务理解、知识增强和流程调度中心。当前已有的分层强化学习模型仍是技术核心，RAG、安全约束和视景模块则负责提升系统完整性、可解释性和论文展示效果。

从硕士毕设工作量看，最稳妥的路线是：先完成 Agent 编排层和 RL 决策封装，再补充轻量 RAG 和安全约束，最后用 Tacview 和实验指标完成验证。这样既能体现“智能体”热点，又不会偏离当前项目已有成果。
