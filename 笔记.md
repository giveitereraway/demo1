### 本项目创新点
1、智能体通过GRU网络维护历史状态信息，能够处理部分可观测环境下的序列决策问题，表现出对历史观测数据的有效利用能力
2、采用PPO算法结合深度神经网络，智能体能够从经验中持续学习和优化策略，自适应各种空战环境条件。
3、代码中实现了先验知识与数据驱动学习的结合机制（ use_prior 参数），使智能体能够融合领域专家知识与自主学习能力。
4、项目支持分层强化学习架构，智能体能够实现高层战术决策与低层飞行控制的有效分离与协同。
5、通过自博弈训练机制和ELO评分系统，智能体能够持续迭代进化，不断提升空战能力。
6、基于JSBSim物理引擎构建的环境中训练，智能体具有对真实物理约束的良好适应性。
7、项目支持1v1、2v2等多种任务场景，智能体具有较强的任务适应性和泛化能力。

### 智能体训练流程
- 1.环境初始化 → 2. 经验收集 → 3. PPO训练 → 4. 模型评估 → 5. 实时监控 → 6. 模型保存 → 7. 性能优化

                                        训练过程的完整工作流程可以概括为：

                                        1. 数据准备阶段 ：
                                        - 通过 collect 和 rollout 方法收集经验数据并存储到 buffer
                                        - 调用 compute 方法计算回报和优势函数
                                        2. 训练阶段 （由 train 方法执行）：
                                        - 策略进入训练模式
                                        - 执行多轮 PPO 更新
                                        - 更新缓冲区状态
                                        - 返回训练指标
                                        3. 迭代优化 ：
                                        - 训练完成后继续收集新的经验数据
                                        - 重复上述过程直到达到预定的训练步数

- 奖励函数得到的奖励 主要用于计算优势函数

- 智能体训练过程中task的作用：
    - 定义观测空间、动作空间
    - 状态观测转换，将JSBSim的原始飞行数据转换为PPO算法可处理的标准化观测
    - 动作归一化，将PPO输出的离散动作转换为JSBSim可执行的连续控制指令
    - 设置奖励函数、终止条件

- 智能体训练过程中 use-selfplay 的作用：
    - 无selfplay：智能体仅基于环境反馈学习，不与其他智能体互动。
    - 有selfplay：智能体在训练过程中与自身的旧版本智能体互动，通过对比优势函数来学习。
    JSBSimRunner（基础运行器） ：
        - 单智能体训练 ：只训练一个智能体策略
        - 简单对抗 ：对手通常是固定策略或简单规则
        - 数据收集 ： `collect` 方法只收集当前智能体的动作
        - 经验回放 ： `ReplayBuffer` 存储单个智能体的经验 
    SelfplayJSBSimRunner（自博弈运行器） ：
        - 多策略对抗 ：维护一个策略池，智能体与历史版本对抗
        - 对手管理 ：支持多个对手策略，可动态切换
        - ELO评分系统 ：为每个策略版本计算ELO评分
        - 复杂数据收集 ： `collect` 方法同时收集智能体和对手的动作

- 智能体训练分层（Hierarchical）的作用：
        高层策略训练流程：
        1. 高层策略接收环境观测
        2. 高层策略输出高层动作（3维MultiDiscrete空间）
        3. 通过 normalize_action 方法将高层动作转换为底层动作
        4. 执行底层动作并获取环境反馈
        5. 基于环境反馈更新高层策略参数
1. 动作空间设计差异
    非分层任务（SingleCombatTask）：
        - 动作空间： spaces.MultiDiscrete([41, 41, 41, 30]) - 4维离散动作
        - 直接控制飞机的4个执行器：副翼、升降舵、方向舵、油门
        - 动作维度：41×41×41×30 = 约200万个离散动作
    分层任务（HierarchicalSingleCombatTask）：
        - 动作空间： spaces.MultiDiscrete([3, 5, 3]) - 3维离散动作,对应高层飞行控制
                    spaces.Discrete(2) 对应 导弹发射决策 ，为二元选择（0=不发射, 1=发射）
        - 高层动作：控制飞机的宏观状态变化
            - 高度变化：[-0.1, 0, 0.1] km
            - 航向变化：[-π/6, -π/12, 0, π/12, π/6] rad
            - 速度变化：[-0.05, 0, 0.05] Mach
        - 动作维度：3×5×3 = 45个离散动作

2. 动作转换机制
    - 高层策略 ：学习选择宏观动作（高度/航向/速度变化）
    - 低层策略 ：预训练的基线模型（BaselineActor）将高层动作转换为底层控制指令
    - 动作转换流程 ：
        高层策略输出[3,5,3]动作
        构建12维输入观测（高层动作 + 当前状态）
        低层策略输出4维连续控制指令
        归一化为[-1,1]范围的执行器控制值
3. 训练复杂度对比
    非分层任务：
        - 需要学习从状态直接映射到精细控制指令
        - 动作空间大，训练难度高
        - 需要大量样本学习飞行控制技能
    分层任务：
        - 将问题分解为两个层次：
            - 高层：战术决策（去哪里、怎么飞）
            - 低层：飞行控制（如何执行）
        - 利用预训练的低层策略，减少学习负担
        - 更适合学习复杂的战术行为
4. 在导弹任务中的扩展
    非分层导弹任务（SingleCombatShootMissileTask）：
        - 动作空间： spaces.Tuple([spaces.MultiDiscrete([41,41,41,30]), spaces.Discrete(2)])
        - 同时学习飞行控制和导弹发射决策
    分层导弹任务（HierarchicalSingleCombatShootTask）：
        - 动作空间： spaces.Tuple([spaces.MultiDiscrete([3,5,3]), spaces.Discrete(2)])
        - 高层决策：战术机动 + 发射时机
        - 低层执行：飞行控制由基线模型处理

- 智能体训练过程中 use-eval 的作用：
    - 无eval：仅在训练过程中评估智能体性能，不进行实时监控。
    - 有eval：定期评估智能体在评估环境中的性能，提供实时反馈。在Runner的eval()方法中使用
    - --eval-interval 1 ：每1个训练回合进行一次评估  --eval-episodes 1 ：每次评估运行1个完整回合  --n-eval-rollout-threads 1 ：使用1个并行环境进行评估



### gymnasium.spaces库里的几类空间类型
- Box：连续空间，定义了每个维度的上下界，例如状态空间或动作空间。
    例如：
        space = spaces.Box(low=-10, high=10, shape=(3,))  # 三维连续空间，每个维度的取值范围为[-10, 10]
- Discrete：具有有限个可能取值的离散空间，例如动作空间。
    例如：
        space = spaces.Discrete(4)  # 4种离散状态
- MultiDiscrete：多离散空间，用于表示离散状态或动作的向量，每个元素可以取多个离散值。
    例如：
        space = spaces.MultiDiscrete([4, 2, 3])  # 第一个维度有4种离散状态，第二个维度有2种离散状态，第三个维度有3种离散状态
- MultiBinary：多二进制空间，用于表示多个二值状态的向量，每个元素只能取0或1。
    例如：
        space = spaces.MultiBinary(5)  # 5个二值状态
- Tuple：元组空间，用于表示固定长度的有序元素集合，每个元素可以是不同的空间类型。
    例如：
        space = spaces.Tuple([spaces.MultiDiscrete([3, 5, 3]), spaces.Discrete(2)]) # 第一个元素是3维离散动作，第二个元素是2种离散状态
- Dict：字典空间，用于表示观测空间或动作空间的复杂结构，其中每个键对应一个子空间。
    例如：
        space = spaces.Dict({
            'position': spaces.Box(low=-10, high=10, shape=(3,)),  # 三维位置
            'velocity': spaces.Box(low=-5, high=5, shape=(3,)),   # 三维速度
            'status': spaces.Discrete(4)  # 4种离散状态
        })


### 位置参数和关键字参数的区别
- 位置参数：按照参数定义的顺序传递的参数，调用时不需要指定参数名。
    - 必须按照定义时的顺序传递
    - 不能跳过任何必需的位置参数
    - 数量必须匹配
- 关键字参数：在调用函数时，通过指定参数名来传递参数，调用时可以不按照参数定义的顺序传递。
    - 不依赖于参数顺序
    - 可以跳过某些参数（如果有默认值）

### 解包操作符 * 和 ** 的区别
- * 解包操作符用于将可迭代对象（如列表、元组、字符串等）解包为单独的元素。解包为位置参数
- ** 解包操作符用于将字典解包为关键字参数。
